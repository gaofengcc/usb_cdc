## 工作流程

### 模式1 — 学习模式

1. 用户把 ESP32 串在 PC 下载工具和目标芯片之间
2. ESP32 I2S 总线以并行采样模式监听 TX/RX 两路信号
3. 内置波形引擎自动识别：
   - 波特率（PCNT 计数法）
   - 数据位数 / 校验位 / 停止位
   - RTS/DTR 复位脉冲宽度
4. 完整记录二进制数据流 + 每帧时间戳 → 保存到 SD/Flash
5. 导出一个"协议模板"（JSON 格式）

### 模式2 — 下载模式

1. 用户选择已学习的协议模板
2. 接上目标芯片的 TX/RX/RTS/DTR
3. ESP32 通过 RMT/UART 逐帧回放
4. 每帧等 ACK（如目标芯片的 0x79/0x1F），超时重试
5. 完成下载 → 校验 → 报告结果

### 模式3 — 协议代理（高速协议）

1. ESP32 跑 CMSIS-DAP v2 固件
2. USB 连 PC，目标接 ESP32 的 SWD/JTAG 引脚
3. 电脑直接用 OpenOCD/PyOCD 操作目标芯片
4. 一根 USB-C 搞定下载调试，不用再买 JLink

## 硬件参考

| 组件 | 说明 | 成本 |
|------|------|------|
| ESP32-S3 | 双核 240MHz，USB-OTG，足够快 | ~¥25 |
| TXS0108 | 8 路双向电平转换，覆盖 TX/RX/RTS/DTR + 备用 | ~¥3 |
| Micro SD 卡槽 | 存储协议模板和固件 | ~¥2 |
| USB-C | 供电 + 数据通信 | ~¥1 |
| TVS 二极管 | 保护 GPIO 不被烧 | ~¥0.5 |

**BOM 总计：约 ¥35（含 PCB）**

## 输出物

1. **ESPTool-Learner** — 学习引擎固件
2. **ESPTool-Player** — 回放引擎固件（含已学习模板）
3. **ESPTool-CMSIS** — CMSIS-DAP 代理固件
4. **PC 工具** — 模板管理 + 协议编辑 GUI（可选）

## 开发路线

1. **MVP（UART 学习 + 回放）** → 先让 ESP32 学会烧 ESP32
2. **增加 I2C/SPI** → 覆盖 STM32 等
3. **CMSIS-DAP 代理** → 覆盖 ARM 芯片
4. **模板编辑器和社区共享** → 大家上传/分享芯片下载模板
