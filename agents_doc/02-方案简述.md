## 工作流程

### 模式1 — 学习模式

1. 用户让下载工具连接到 ESP32 暴露的串口入口, 或连接到 PC 侧虚拟 COM 代理
2. 代理层将上位机数据透传到目标芯片, 同时记录 TX/RX, RTS/DTR, 串口参数和关键时间戳
3. 学习引擎按会话生成协议模板:
   - 串口参数
   - 复位控制时序
   - 命令帧 / 应答帧
   - 帧间延时统计
4. 模板保存到本地或导出给 ESP32
5. ESP32 后续按模板独立执行下载流程

### 模式2 — 下载模式

1. 用户选择已学习的协议模板
2. 接上目标芯片的 TX/RX/RTS/DTR
3. ESP32 通过 UART/GPIO 逐帧回放
4. 每帧等 ACK（如目标芯片的 0x79/0x1F），超时重试
5. 完成下载 → 校验 → 报告结果

### 模式3 — 被动监听 / 高速协议代理（后续）

1. 补充 I2S / IIS 被动监听能力, 用于旁路学习或疑难分析
2. 针对 SWD/JTAG 等协议评估代理转发方案
3. 电脑直接用 OpenOCD/PyOCD 操作目标芯片
4. 作为主功能稳定后的扩展方向

## 硬件参考

| 组件 | 说明 | 成本 |
|------|------|------|
| ESP32-S3 | 双核 240MHz，USB-OTG，足够快 | ~¥25 |
| TXS0108 | 8 路双向电平转换，覆盖 TX/RX/RTS/DTR + 备用 | ~¥3 |
| Micro SD 卡槽 | 存储协议模板和固件 | ~¥2 |
| USB-C | 供电 + 数据通信 | ~¥1 |
| TVS 二极管 | 保护 GPIO 不被烧 | ~¥0.5 |

硬件 BOM 总计: 约 ¥35（含 PCB）

## 输出物

1. **ESPTool-Learner** — 学习引擎固件
2. **ESPTool-Player** — 回放引擎固件（含已学习模板）
3. **COM / 串口代理工具** — 学习记录与模板导出
4. **ESPTool-CMSIS** — 后续高速协议代理固件
5. **PC 工具** — 模板管理 + 协议编辑 GUI（可选）

## 开发路线

1. **MVP（串口代理学习 + ESP32 回放）** → 先跑通 ESP32 / STM32 等 UART Bootloader
2. **完善模板和回放状态机** → 覆盖更多 ACK/重试场景
3. **增加 I2C/SPI 或 I2S 被动监听** → 作为扩展能力
4. **CMSIS-DAP 代理** → 覆盖 ARM 芯片
5. **模板编辑器和社区共享** → 大家上传/分享芯片下载模板
