# ESP32 通用 UART 下载工具 — 详细方案

> 聚焦 UART 串口下载场景，覆盖主流芯片的 Bootloader 烧录。

---

## 1. 覆盖目标芯片（UART Bootloader 类）

| 芯片系列 | 下载协议 | 波特率 | 特殊信号 | 备注 |
|----------|---------|--------|---------|------|
| **ESP32 / ESP8266** | ESP 下载协议 | 115200~921600 | GPIO0 (低电平) + EN (复位脉冲) | 最简单，可自举验证 |
| **STM32 (系统存储器)** | STM32 UART Bootloader | 115200 (默认) | BOOT0=1 + 复位 | AN3155 协议，命令应答式 |
| **NXP LPC** | ISP UART | 9600~115200 | ISP pin + 复位 | 字符握手 |
| **TI CCxx** | Serial Bootloader | 115200 | 专用 IO 触发 | 各系列不同 |
| **GD32 / AT32** | 类 STM32 协议 | 115200 | BOOT0 控制 | 兼容 STM32 命令集 |
| **联发科 / 全志** | UART Download | 115200~921600 | 上电时序特殊 | 私有协议，需学习 |

## 2. 学习引擎核心原理（I2S 并行采样）

### 为什么用 I2S 而不是 UART 硬件直连？

学习模式的核心矛盾：**不知道波特率、不知道数据格式**，所以不能用硬件 UART 去解。

### I2S 做逻辑分析仪

ESP32 的 I2S 外设在 **RX 模式**下可以当并行采样器用：

```
I2S 时钟 = APLL 分频，最高约 8MHz 采样率
I2S 数据线 = GPIO，可同时采样 2~4 路数字信号
```

#### 采样过程

```
PC 下载工具 ──TX──→ ESP32 GPIO_A (I2S Data0)
目标芯片 RX ←─────────
                      ESP32 GPIO_B (I2S Data1) ←── 目标芯片 TX
                      
I2S 同时采样 A/B 两路 @ 最高 8MHz
时间分辨率: 125ns
```

### 自动识别算法

```
原始比特流 (8MHz 采样)
    ↓
1. 边沿检测 → 找到 TX 信号跳变
2. 最小脉冲宽度测量 → 推算波特率
   - 单比特宽度 = 最短低电平脉冲宽度（起始位+0）
   - 波特率 = 1 / 单比特宽度
3. 按推算波特率解帧：
   - 起始位 (1bit low) + 数据位 (8/9bit) + 校验位(可选) + 停止位(1/2bit high)
4. 帧同步验证：
   - 检查停止位是否 stable high
   - 连续 3 帧通过 → 锁定配置
5. RTS/DTR 信号独立通道识别：
   - I2S 额外通道采样 RTS/DTR
   - 测量复位脉冲宽度
```

#### 精度评估

| 波特率 | 比特宽度 | I2S@8MHz 采样点 | 识别精度 |
|--------|---------|----------------|---------|
| 9600 | 104.17μs | ~833 个采样点 | ★★★★★ |
| 115200 | 8.68μs | ~69 个采样点 | ★★★★★ |
| 921600 | 1.085μs | ~8.7 个采样点 | ★★★★ |
| 2Mbps | 0.5μs | ~4 个采样点 | ★★★ |
| 4Mbps | 0.25μs | ~2 个采样点 | ★★ 勉强 |

**结论：2Mbps 以下完全可靠，这是 UART 下载的主流区间。**

## 3. 回放引擎（下载模式）

### 两种回放策略

#### 策略 A — 纯波形回放（RMT 模式）

```
RMT 存储从 I2S 学习到的完整 TX 波形
→ 直接输出到目标芯片 RX 引脚
→ 一次性精确复现，无任何协议理解
```

- ✅ 简单粗暴，不需要理解协议语义
- ❌ 没有 ACK 校验，失败时无法做智能重试
- ❌ 下载器如果有随机延迟间隙，回放可能歪

**适用于：** 协议已充分理解，已知芯片无 ACK/超时限制

#### 策略 B — 语义回放（推荐）

```
I2S 学习 → 提取二进制帧序列 + 帧间距统计
          → 按帧回放，每帧用硬件 UART 发送
          → 接收芯片 ACK，匹配期望响应
          → 超时重试当前帧
```

- ✅ 有 ACK 检验
- ✅ 时序自适应（用统计中位帧间距，不是固定值）
- ✅ 失败时可精确报错到第几帧
- ❌ 需要理解协议命令-响应对

**适用于：** 所有带 ACK 的 UART Bootloader（STM32、NXP、GD32 等）

## 4. 信号引脚需求

| ESP32 GPIO | 功能 | 备注 |
|-----------|------|------|
| **GPIO 3** (RX0) | UART0 RX | 用于学习模式接 PC TX（硬件 UART 辅助识别） |
| **GPIO 1** (TX0) | UART0 TX | 学习模式接 PC RX（记录 PC 收到什么） |
| **GPIO 4** | I2S 通道 A | 学习模式接 PC TX 信号 |
| **GPIO 5** | I2S 通道 B | 学习模式接目标芯片 TX 信号 |
| **GPIO 6** | RTS 捕获 | 学习模式接 PC DTR/RTS |
| **GPIO 7** | DTR 捕获 | 学习模式接 PC RTS/DTR |
| **GPIO 10** | UART1 TX → 目标 RX | 下载模式输出 |
| **GPIO 11** | UART1 RX ← 目标 TX | 下载模式接收 ACK |
| **GPIO 12** | 目标 EN/RST | 下载模式控制复位 |
| **GPIO 13** | 目标 BOOT/GPIO0 | 下载模式控制启动模式 |
| **GPIO 14** | LED 指示 | 状态指示灯 |
| **GPIO 15** | 模式按键 | 切换学习/下载模式 |

**电平转换：** 所有 GPIO 通过 TXS0108 双向电平转换芯片引出，支持 3.3V 和 5V 目标芯片。

## 6. 第一代产品规格

```
ESP32 Universal UART Flasher v1

硬件：ESP32-S3 + TXS0108 + SD卡 + USB-C + 按键x1 + LEDx1
尺寸：约 60x30mm（比火柴盒略大）
接口：6pin 排针（VCC/TX/RX/RTS/DTR/GND）

支持协议：
  - 标准 UART（7/8 bit, 偶/奇/无校验, 1/2 stop bit）
  - 波特率自动识别范围：9600 ~ 2Mbps
  - RTS/DTR 自动复位信号学习+回放

工作模式：
  ① LEARN — 串在 PC 下载器和目标之间，偷学下载时序
  ② FLASH — 接入目标芯片 TX/RX，直接下载
  ③ PASS — 透传模式，当 USB↔UART 转接板用

存储：SD卡存储多个协议模板，支持命名和选择

供电：USB-C 5V 或目标板取电（3.3V/5V）
```
