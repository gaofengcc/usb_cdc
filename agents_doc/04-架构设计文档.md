## 2. 模块定义

### 2.1 硬件抽象层 (HAL)

#### 2.1.1 I2S 采样器 (`i2s_sampler.h`)

```
职责：
  - 初始化 I2S 为并行 RX 模式
  - 在 DMA 缓冲到达时回调通知
  - 提供采样率配置（自适应）

接口：
  struct sampler_config {
      uint32_t sample_rate;        // 采样率 (Hz)
      uint8_t  channel_count;      // 通道数 (1~4)
      uint8_t  gpio_pins[4];       // GPIO 映射
      size_t   dma_buffer_size;    // DMA buffer 大小 (bytes)
      bool     oneshot;            // 单次采集 / 持续采集
  };

  esp_err_t sampler_init(sampler_config_t *cfg);
  esp_err_t sampler_start(void);
  esp_err_t sampler_stop(void);
  void sampler_set_callback(sampler_cb_t cb);  // cb(data, len, timestamp)

数据流：
  I2S 硬件 → DMA → 环形缓冲区 → 回调 → 协议引擎
```

#### 2.1.2 RMT 脉冲收发器 (`rmt_pulse.h`)

```
职责：
  - 精确生成和捕获脉冲序列
  - 作为 UART 回放的备用策略
  - 测量脉冲宽度（用于波特率测量）

接口：
  esp_err_t rmt_tx_raw(const rmt_item32_t *items, size_t len);
  esp_err_t rmt_rx_start(rmt_rx_cb_t cb);

数据格式：
  rmt_item32_t = { duration0: uint16, level0: uint8,
                   duration1: uint16, level1: uint8 }
  单位: RMT 时钟 tick (通常 12.5ns @ 80MHz)
```

### 2.2 协议引擎层

#### 2.2.1 帧分析器 (`frame_analyzer.h`)

```
职责：
  - 从 I2S 原始采样数据中提取通信帧
  - 自动识别 UART 参数（波特率、数据位、校验位、停止位）
  - 帧同步验证

核心算法：

  raw_samples[8MHz, 2ch]
      │
      ▼
  [边缘检测] → 标记所有 TX 信号跳变点（上升沿/下降沿）
      │
      ▼
  [最小脉冲统计] → 统计所有低电平脉冲宽度分布
      │              波特率 = 1 / 脉冲宽度分布峰值的众数
      ▼
  [帧同步锁定] → 按候选波特率解帧，验证停止位高电平
      │              连续 3 帧通过 → 锁定 UART 参数
      ▼
  [帧切片] → 按帧分割原始数据，输出 frame_t 序列
      │
      ▼
  [RTS/DTR 分析] → 提取复位信号的脉冲宽度和时序关系

输出：
  struct uart_params {
      uint32_t baudrate;           // 识别到的波特率
      uint8_t  data_bits;          // 7 or 8
      uart_parity_t parity;        // NONE/ODD/EVEN
      uart_stop_bits_t stop_bits;  // 1 or 2
      bool     valid;              // 是否通过帧同步验证
      float    confidence;         // 置信度 0.0~1.0
  };

  struct frame_t {
      uint64_t timestamp_us;       // 帧起始时间戳
      uint16_t frame_len;          // 帧长度（字节）
      uint8_t  data[MAX_FRAME];    // 帧数据
      uint32_t interframe_gap_us;  // 与上一帧的时间间隔
      bool     has_ack;            // 是否有 ACK 响应
      uint16_t ack_len;            // ACK 长度
      uint8_t  ack_data[MAX_ACK];  // ACK 内容
      uint64_t ack_delay_us;       // 发送到收到 ACK 的延迟
  };
```

#### 2.2.2 学习引擎 (`learner.h`)

```
职责：
  - 协调 I2S 采样 + 帧分析
  - 记录完整的下载会话
  - 生成可导出的协议模板

状态：
  LEARNER_IDLE → LEARNER_LISTENING → LEARNER_CAPTURING → LEARNER_COMPLETE

输出（最终保存为 JSON 模板）：
  struct protocol_template_t {
      char         chip_name[32];       // 芯片名称（用户命名）
      char         tool_name[32];       // 下载工具名称（自动识别?）
      uart_params_t uart;               // UART 参数
      uint32_t     total_frames;        // 总帧数
      
      // 复位时序
      struct {
          bool     has_rts;             // 是否有 RTS 控制
          bool     has_dtr;             // 是否有 DTR 控制
          uint32_t rts_pulse_us;        // RTS 脉冲宽度
          uint32_t dtr_pulse_us;        // DTR 脉冲宽度
          uint32_t reset_to_first_frame_delay_us; // 复位后到第一帧的延迟
          uint8_t  reset_logic;         // 复位信号有效电平
      } reset;
      
      // 帧序列
      frame_t    *frames;              // 帧序列指针
      uint64_t    total_duration_us;   // 总下载时长
      
      // 元信息
      uint64_t    learn_timestamp;     // 学习时间戳
      uint32_t    crc32;               // 模板校验
  };
```

#### 2.2.3 回放引擎 (`player.h`)

```
职责：
  - 加载协议模板
  - 控制下载流程：复位 → 发帧 → 等 ACK → 下一帧
  - 错误处理和重试

状态机：
  PLAYER_IDLE → PLAYER_RESET → PLAYER_SYNC → PLAYER_SENDING
  → PLAYER_WAIT_ACK → (循环) → PLAYER_DONE / PLAYER_ERROR

核心逻辑：
  while (frame_idx < template->total_frames) {
      // 1. 发送当前帧
      uart_write_bytes(template->frames[frame_idx].data, len);
      
      // 2. 等待 ACK（如有）
      if (frame->has_ack) {
          int ret = uart_read_ack(frame->ack_data, frame->ack_len, 
                                  timeout = frame->ack_delay_us * 2);
          if (ret == ESP_ERR_TIMEOUT) {
              retry_count++;
              if (retry_count > MAX_RETRY) {
                  report_error(FRAME_N_TIMEOUT, frame_idx);
                  break;
              }
              // 重新发送（可能需要重新复位）
              goto retry_frame;
          }
      }
      
      // 3. 遵守帧间距
      delay_us(frame->interframe_gap_us);
      frame_idx++;
  }
```

### 2.3 应用层

#### 2.3.1 学习管理器 (`learn_ctrl.h`)

```
职责：
  - 管理一次完整的学习会话
  - 进度反馈（LED 闪烁模式）
  - 模板命名与存储

流程：
  1. 用户进入 LEARN 模式（按键 + LED 指示）
  2. 等待检测到 TX 信号活动（PC 开始下载）
  3. FrameAnalyzer 自动识别参数并锁定
  4. Learner 记录全部帧序列 + ACK
  5. 信号静默 5 秒 → 认为下载结束
  6. 提示用户命名芯片（LED 模式指示）
  7. 保存模板到 SD 卡
```

#### 2.3.2 下载管理器 (`flash_ctrl.h`)

```
职责：
  - 从 SD 卡选择协议模板
  - 控制下载流程
  - 显示下载进度（LED）

流程：
  1. 用户进入 FLASH 模式
  2. 轮流闪烁列出可用模板（根据按键选择）
  3. 选定模板 → Player 加载
  4. 执行复位时序
  5. 逐帧发送 + 等待 ACK
  6. 完成 → LED 常亮 + 可拔除
  7. 失败 → LED 快闪 + 可重试
```

## 3. 状态机设计

### 3.1 系统主状态机

```
                     ┌──────────┐
       上电─────────→│  BOOT    │
                     └────┬─────┘
                          │ 硬件初始化完成
                          ▼
                     ┌──────────┐
         ┌──────────│  READY   │──────────┐
         │          └────┬─────┘          │
         ▼               ▼                ▼
   ┌──────────┐   ┌──────────┐    ┌──────────┐
   │  LEARN   │   │  FLASH   │    │PASSTHRU  │
   └────┬─────┘   └────┬─────┘    └────┬─────┘
        │               │               │
        │ 完成/取消      │ 完成/取消      │ 退出
        └───────┬───────┴───────┬───────┘
                ▼               ▼
           ┌──────────┐   ┌──────────┐
           │  SLEEP   │   │  ERROR   │
           └──────────┘   └────┬─────┘
                                │ 按键复位
                                ▼
                            ┌──────────┐
                            │  READY   │
                            └──────────┘
```

**模式切换方式：**
- 上电默认：READY（无任务，低功耗等待）
- 按模式键 < 1s：轮换 LEARN → FLASH → PASSTHRU
- 长按 > 3s：返回 READY
- 10 分钟无操作：SLEEP

### 3.2 学习子状态机

```
LEARN_IDLE ──→ LEARN_WAIT_SIGNAL ──→ LEARN_ANALYZE ──→ LEARN_CAPTURE
                    ▲                                        │
                    │                         信号静默 > 5s   │
                    │                                        ▼
                    │                               LEARN_COMPLETE
                    │                                     │
                    │                                     ▼
                    │                              LEARN_SAVE_DONE
                    └─────────────────────────────────────────┘
                        (按键取消，重新等待)
```

| 状态 | 动作 | 条件 | 超时 |
|------|------|------|------|
| IDLE | LED 慢闪，等待用户开始 | 按键确认 | - |
| WAIT_SIGNAL | 启动 I2S 监听模式，检测 TX 是否有活动 | TX 连续 10ms 有跳变 | 60s→回 IDLE |
| ANALYZE | 检测到活动→锁定波特率/帧参数 | 连续 3 帧同步通过 | 2s→回 WAIT |
| CAPTURE | 完整记录所有帧 + ACK + 复位信号 | TX 静默 > 5s | 最大 30min |
| COMPLETE | 提示用户命名 | 按键确认或串口输入 | 30s→自动命名 |
| SAVE_DONE | 保存模板到 SD，返回 READY | SD 写入完成 | - |

### 3.3 下载子状态机

```
FLASH_IDLE ──→ FLASH_SELECT ──→ FLASH_CONNECT ──→ FLASH_RESET
                                                      │
                                                      ▼
                                              FLASH_SENDING_FRAME
                                                      │
                                                      ▼
                                              ┌─ FLASH_WAIT_ACK ──┐
                                              │                    │
                                          ACK匹配            超时重试>3次
                                              │                    │
                                              ▼                    ▼
                                       FLASH_NEXT_FRAME    FLASH_RETRY_RESET
                                              │                    │
                                         帧发完了             重试>2次
                                              │                    │
                                              ▼                    ▼
                                       FLASH_DONE          FLASH_ERROR
```

| 状态 | 动作 | 条件 |
|------|------|------|
| IDLE | 列出可用模板 | - |
| SELECT | 按键翻页选择模板 | 选中后确认 |
| CONNECT | 设置 UART 参数、电平电压 | 完成配置 |
| RESET | 发出复位脉冲序列 | 脉冲发送完成 |
| SENDING_FRAME | 发送当前帧数据 | 帧发送完成 |
| WAIT_ACK | 等待目标芯片 ACK | 超时或收到数据 |
| RETRY_RESET | 重新复位芯片 | 再次尝试发送 |
| DONE | 校验完成→LED 常亮 | 成功 |
| ERROR | LED 快闪→显示错误码 | 失败 |

## 6. 任务调度（FreeRTOS）

### 6.1 任务划分

| 任务名 | 优先级 | 栈大小 | 职责 | 周期/触发 |
|-------|-------|--------|------|----------|
| `app_task` | 8 (高) | 4096 | 应用层状态机、模式切换、按键处理 | 事件驱动 |
| `protocol_task` | 6 (中) | 4096 | 学习/回放引擎运行 | 状态机驱动 |
| `led_task` | 3 (低) | 1024 | LED 闪烁模式、进度指示 | 定时 50ms |
| `storage_task` | 5 (中) | 2048 | SD 卡读写、JSON 序列化 | 事件驱动 |
| `console_task` | 4 (低) | 2048 | UART0 控制台交互（调试用） | 串口中断 |

### 6.2 任务间通信

```
                  按键 GPIO 中断
                       │
                       ▼ ── 消息队列 (xQueue)
                 ┌──────────┐
                 │ app_task │
                 └────┬─────┘
                      │
         ┌────────────┼────────────┐
         │            │            │
         ▼            ▼            ▼
  ┌───────────┐ ┌──────────┐ ┌──────────┐
  │protocol   │ │storage   │ │led_task  │
  │_task      │ │_task     │ │          │
  └───────────┘ └──────────┘ └──────────┘

通信机制：
  事件队列: xQueueSend / xQueueReceive
  标志位:   xEventGroupSetBits / xEventGroupWaitBits
  I2S DMA:  直接在 ISR 回调中通过 xQueueSendFromISR 送到 protocol_task
```

### 6.3 关键时序约束

```
I2S DMA 中断频率：
  采样率 8MHz × 2ch × 16bit = 32Mbps
  DMA buffer 1024 bytes → 每 256μs 触发一次 DMA 完成中断
  ISR 开销必须 < 100μs（纯数据搬移，不做协议解析）

帧发送精度：
  UART 帧间距精度要求：±50μs（用 esp_timer 或定时器组实现）
  RMT 策略可达到 ±1μs

复位脉冲精度：
  STM32 典型复位脉冲 10ms，误差 ±1ms 即可
  ESP32 的 EN 拉低需 100ms，要求较低
```

## 7. 存储布局

### 7.1 ESP32 Flash 分区

| 分区名 | 偏移 | 大小 | 用途 |
|--------|------|------|------|
| nvs | 0x9000 | 20KB | Wi-Fi/校准数据（不用） |
| otadata | 0xE000 | 8KB | OTA 启动选择 |
| app0 | 0x10000 | 1.5MB | 主固件 |
| app1 | 0x1F0000 | 1.5MB | OTA 备份（可放模板缓存） |
| spiffs | 0x3D0000 | 1MB | 配置 + 默认协议模板缓存 |
| nvs_key | 0x4D0000 | 4KB | 加密密钥（预留） |

### 7.2 运行时内存

```
IRAM (代码 + 关键数据):
  - I2S DMA buffer A:  4096 bytes  (固定映射)
  - I2S DMA buffer B:  4096 bytes  (固定映射)
  - RMT TX/RX buffer:  2048 bytes
  - FreeRTOS 内核:     ~8KB

DRAM (运行时数据):
  - 帧分析器工作区:    8KB
  - 协议模板帧缓存:    ~4KB (最多存 512 帧的单帧缓存)
  - UART 收发 buffer:  2KB x2
  - 应用状态:          2KB

PSRAM (如果使用 ESP32-S3 + PSRAM):
  - 完整帧序列缓冲区:  最大 2MB（支持超大下载会话分析）
  - 固件数据暂存:     ~1MB
```

### 9.2 上报策略

```
LED 指示灯编码：
  慢闪 (1Hz):   等待操作
  快闪 (5Hz):   学习中（捕获阶段）
  呼吸灯:       下载进行中
  常亮:         完成
  3 次快闪:     出错（重复显示错误码）

  eg. ERR_FLASH_NO_ACK (201)
  → 快闪 2 次（百位 2）
  → 停顿 500ms
  → 快闪 0 次（十位 0）
  → 停顿 500ms
  → 快闪 1 次（个位 1）
  → 停顿 2s 后重复
```

## 10. 边界情况

### 10.1 已知边界情况及处理

| 情况 | 处理 |
|------|------|
| **波特率自动识别失败** | 回退到用户手动配置（通过按键组合或 PC 工具设置） |
| **多帧连续无 ACK** | 不立即放弃，执行 3 次重试（含重新复位），仍失败才报错 |
| **学习过程中 PC 下载工具多次复位重试** | 检测到复位信号 → 标记 session 分节 → 只保留最后一次完整序列 |
| **空模板（芯片不支持下载或下载失败）** | 学习时检查总帧数 < 5 → 提示"未检测到有效的下载通信" |
| **SD 卡未插入** | 降级到 SPI Flash 存储，限制存储 3 个模板，提示用户插卡 |
| **模板兼容性** | 同一型号不同批次芯片可能有 ACK 延迟差异 → 回放时使用自适应超时（2x 学习值） |
| **下载一半断电** | 下一次启动自动检查 SD 卡上的临时文件，标记为无效并提示 |
| **TX/RX 接反** | 学习阶段识别到无有效帧 → 提示"检查 TX/RX 是否接反" |
| **波特率不匹配** | 回放时连续 5 帧无 ACK 且非超时 → 猜测波特率不对，建议重新学习 |
| **纯下载数据零填充** | 空字节段长尾 → 不写模板，避免模板膨胀 |

### 10.2 缓冲区管理

```
I2S DMA 双缓冲策略：
  缓冲 A:  正在填充（I2S 硬件写入）
  缓冲 B:  正在处理（CPU 读取分析）
  
  切换条件：DMA 完成中断
  
  反压处理：
    如果 CPU 处理慢于 DMA 填充速度 → 丢弃旧数据
    （学习模式允许丢部分数据，只要关键帧偶现即可锁定参数）
    
    下载模式：反压不适用（数据由 CPU 主动控制）
```
