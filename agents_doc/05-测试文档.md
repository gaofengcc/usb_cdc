> - P0: 串口代理学习, 模板生成, UART/GPIO 回放主链路.

### 4.1 Session Parser 测试 (test_session_parser.c)

#### TC-FA-001: 标准 115200-8N1 波特率识别

```
测试类型:  单元测试（Host）
优先级:   P0
TDD 阶段: RED → GREEN

输入:
  合成 UART 信号，115200 baud，8N1
  数据内容: 0x55 (01010101) 连续帧，间隔 500μs
  采样率: 8MHz
  共 10 帧

期望输出:
  frame_analyzer_get_params().baudrate == 115200
  frame_analyzer_get_params().data_bits == 8
  frame_analyzer_get_params().parity == NONE
  frame_analyzer_get_params().stop_bits == 1
  frame_analyzer_is_locked() == true
  检测到 10 帧，每帧 1 字节

失败条件（RED 验证）:
  未实现前 → 返回 baudrate=0, is_locked=false
```

#### TC-FA-011: 半双工回显检测

```
测试类型:  单元测试（Host）
优先级:   P1

输入:
  CH0 (PC TX): 发送命令帧
  CH1 (Target TX): 回显相同数据（目标芯片 echo 模式）
  115200-8N1
  3 轮 命令→回显 交互

期望输出:
  每一轮: frames[2n].data == frames[2n+1].data (TX发送 == TX回显)
  frames[2n].ack == 目标回显帧
  即第 0 帧的 ACK 数据 = 第 1 帧（目标 TX 的内容）
```

测试类型:  单元测试（Host）

期望输出:

```

#### TC-FA-013: 波特率容差边界

```
测试类型:  单元测试（Host）
优先级:   P1

输入:
  真实波特率偏差:
  A) 标称 115200，实际 114800 (-0.35%)
  B) 标称 115200，实际 115600 (+0.35%)
  C) 标称 115200，实际 112000 (-2.8%)  ← 边界外

期望输出:
  A) 识别 114800 或 115200 (±3% 误差内可接受)
  B) 识别 115600 或 115200 (±3% 误差内可接受)
  C) 锁定失败或识别到 112000 (±3%)
```

#### TC-FA-014: 帧数据内容逐比特校验

```
测试类型:  单元测试（Host）
优先级:   P0

输入:
  已知内容的 UART 数据:
  帧0: 0x7F
  帧1: 0x00, 0xFF, 0x00, 0xFF, 0x00, 0xFF  (STM32 同步命令)
  帧2: 0x31, 0xCE  (STM32 0x31 命令 + XOR 校验)

期望输出:
  帧1 len==6, data[0]==0x00, data[1]==0xFF, data[2]==0x00, data[3]==0xFF, data[4]==0x00, data[5]==0xFF

测试类型:  单元测试（Host）

期望输出:
  CH0 帧序列: 命令帧
  CH1 帧序列: 应答帧

### 4.3 Learner 测试 (test_learner.c)

> **职责：** 学习引擎核心逻辑（不含 I2S 硬件）

#### TC-LR-001: 单帧学习与帧提取

```
测试类型:  单元测试（Host）
优先级:   P0

输入:
  已分析好的 frame_t 数组（从 frame_analyzer 输出模拟）
  共 5 帧，每帧 1 字节

操作:
  learner_feed_frame(&frame);
  learner_get_template();

期望输出:
  template.total_frames == 5

```

  学习阶段成功识别所有帧

## 7. TDD 工作流

### 7.1 模块开发顺序（按依赖）

```
1. frame_analyzer      ← 最底层，无依赖
2. template_format     ← 无依赖
3. learner              ← 依赖 frame_analyzer + template_format
4. player               ← 依赖 template_format
5. i2s_sampler          ← 硬件依赖，需与 frame_analyzer 联合
6. gpio_control         ← 硬件依赖
7. storage              ← 硬件依赖（可 mock 测试部分逻辑）
8. rmt_pulse            ← 硬件依赖（备用策略）
9. app_ctrl             ← 依赖所有模块
10. 集成测试            ← 全模块联合
```

### 7.2 每个模块的 TDD 流程

```
以 frame_analyzer 为例：

Step 1 — 创建测试文件
  tests/host/test_frame_analyzer/test_frame_analyzer.c
  引入 Unity 框架
  创建 test data 目录, 生成模拟波形

Step 2 — 写第一个测试 (RED)
  TEST_CASE("TC-FA-001: 标准115200-8N1波特率识别", "[frame_analyzer]")
  {
      // 生成 115200-8N1 合成波形
      // 调用 frame_analyzer_feed_samples()
      // 断言 baudrate == 115200
      TEST_ASSERT_EQUAL(115200, frame_analyzer_get_params().baudrate);
  }

Step 3 — 运行 → 验证失败
  cd tests/host && cmake -B build && cmake --build build && ./build/test_frame_analyzer
  → 测试失败，baudrate == 0 不等于 115200 ✓ (RED 确认)

Step 4 — 实现最小代码
  frame_analyzer.c:
  - 脉冲宽度统计 → 计算最小低电平宽度 → 推算波特率

Step 5 — 运行 → 验证通过 (GREEN)
  同上命令
  → 测试通过 ✓

Step 6 — 重构
  检查代码质量，提取公共函数

Step 7 — 下一个测试循环
  TC-FA-002: 9600-8N1 → RED → GREEN → REFACTOR
```

### 7.3 提交规范

```
每次 commit 对应一个 TDD 循环:
  feat(frame_analyzer): 115200-8N1 波特率识别

  RED: test_frame_analyzer/TC-FA-001 — 验证 115200 识别
  GREEN: pulse width statistics → baudrate calculation
  REFACTOR: extract edge detection helper

  Closes: TC-FA-001
```
